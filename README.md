# uniapp-totolist
# frontend-learning-2026

“待办事项”项目：覆盖了前端核心基础界面、交互、数据、状态、存储。

项目需求：
添加任务：输入框 + 添加按钮，回车可添加。
任务列表：展示所有待办事项。
完成任务：点击复选框，标记任务为完成（文字划线或置灰）。
删除任务：每个任务旁有一个删除按钮。
数据统计：显示“还剩 X 项未完成”。
持久化：刷新页面后，任务列表不丢失（使用 localStorage）。
基础样式：干净、居中、有适度间距，移动端响应式。

扩展：
任务编辑（双击修改）
筛选（全部/未完成/已完成）
一键清除所有已完成任务
暗色模式
添加截止日期

项目计划与执行（分 3 天，每天 ≤ 1 小时）
Day 1 | 3月31日  | 结构 & 样式 | 1. 创建页面，搭好基础骨架。2. 写好 CSS（Flex/Grid），完成输入区、任务列表的视觉稿。3. 确保界面在手机和电脑上看起来整洁。
Day 2 | 4月1日 | 交互 & 逻辑 | 1. 用 JavaScript 实现添加/删除/完成任务功能。2. 动态渲染任务列表。3. 实现“剩余未完成”计数。
Day 3 | 4月2日 | 数据持久化 & 收尾 | 1. 接入 localStorage，实现数据保存和加载。2. 修复边界 Bug（如添加空内容）。3. 将项目部署到 GitHub Pages。

技术栈
HTML5：语义化标签
CSS3：Flexbox / Grid，响应式设计
JavaScript (ES6+)：数组操作 (map, filter)，DOM 操作，事件委托，localStorage
